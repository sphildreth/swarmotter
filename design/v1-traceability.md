# v1 capability and acceptance traceability

This matrix is the release-evidence index for `design/requirements.md` and
every completed row in `design/v1-completion-tracker.md`. A completed functional
claim requires all three columns: a reachable production entry, an automated
test that invokes that path, and operator/developer documentation describing the
same behavior. Pure helpers and model-only serialization tests are supporting
evidence, not substitutes for the named production-path tests.

The `v1` name is retained because these are the capabilities originally tracked
for the first complete product line. The assembled release is `v2.0.0` because
ADR-0051 changes the configuration compatibility contract; the capability
scope itself is not reduced.

## Explicit reachability closures

- `TorrentState::TrackerError` is produced only by
  `EngineState::terminal_tracker_error` at real daemon task completion when all
  configured tracker attempts failed and no usable peer/DHT/PEX/webseed signal
  exists. The last tracker failure is retained in `TorrentSummary.error` and
  Torrent Details. `terminal_tracker_failure_sets_tracker_error_and_reannounce_retries`
